- D-01: `ParseArtifact` is one unified top-level envelope for every parse result.
- D-03: `ParseArtifact` reserves stable sections for replay metadata, observed entities/identity, normalized events, aggregate contribution references, diagnostics, and failures.
- D-04: Artifact fields use `snake_case` JSON names and deterministic `BTreeMap` extension maps where ordering is observable.
- D-08: `Diagnostic` records path-based expected shape, observed shape, parser action, and source refs for non-fatal schema drift.
- D-13: Diagnostics and normal artifacts store source coordinates and rule IDs, not raw replay snippets or raw values.
- D-14: `ParseStatus` values are exactly `success`, `partial`, `skipped`, and `failed`.
- D-15: `skipped` is a parser-level status in the artifact envelope; legacy aggregate skip reasons remain diagnostics/compatibility data, not hard failures.
</decision_coverage>
