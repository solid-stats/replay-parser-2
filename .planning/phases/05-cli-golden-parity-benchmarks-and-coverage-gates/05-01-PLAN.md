<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| historical corpus -> committed fixtures | Large, mutable corpus files must not be blindly copied into git. |
| fixture manifest -> test coverage | Manifest metadata must not claim coverage without parser behavior assertions. |
| compatibility fixtures -> canonical identity | Legacy identity behavior remains compatibility evidence, not canonical matching. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | fixture provenance | mitigate | Manifest entries include source, decisions, requirements, and expected behavior. |
| T-05-01-02 | Denial of Service | repository size | mitigate | Committed golden fixtures stay under 50 KB and bulky outputs stay generated/ignored. |
| T-05-01-03 | Repudiation | behavior coverage | mitigate | Tests parse fixtures and assert observable artifact fields for each required edge class. |
</threat_model>
