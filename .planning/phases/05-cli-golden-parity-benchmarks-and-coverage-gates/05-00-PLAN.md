  <action>
    Implement `parse` so it:

    - reads the requested input path into bytes;
    - computes SHA-256 as lowercase hex with `sha2::Sha256`;
    - builds `ReplaySource { replay_id, source_file, checksum }`, where `source_file` is the input path as displayed by the CLI and checksum is `FieldPresence::Present { value: SourceChecksum::sha256(<hex>), source: None }`;
    - uses parser info `{ name: "replay-parser-2", version: env!("CARGO_PKG_VERSION") }`;
    - calls `parser_core::parse_replay(ParserInput { bytes, source, parser, options: ParserOptions::default() })`;
    - writes pretty JSON to the requested `--output` path for success, partial, skipped, and failed artifacts;
    - exits `0` when artifact status is success/partial/skipped and exits non-zero when artifact status is failed or the input/output file operation fails;
    - writes stderr summaries only, with failed parse stderr containing `parse failed:` and unreadable input stderr containing `could not read input:`.

    Add `parse_command.rs` tests using `assert_cmd` and temp directories that:

    - parse `valid-minimal.ocap.json` to an output file and assert JSON contains `"contract_version"`, `"source_file"`, `"checksum"`, and `"replay"`;
    - parse `invalid-json.ocap.json`, assert non-zero exit, assert output JSON contains `"status": "failed"` and `"failure"`, and assert stderr contains `parse failed:`;
    - run the same valid input twice and assert the two output files are byte-identical after normalizing only the chosen output path if the path appears in the artifact.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local filesystem -> parser-core | CLI reads untrusted replay bytes and must not panic or hide structured failures. |
| CLI stderr/stdout -> machine consumers | Machine-readable output must be JSON files/stdout, not ad-hoc stderr text. |
| local adapter -> parser-core | CLI must not add replay selection, identity canonicalization, or server integration behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-00-01 | Tampering | input checksum | mitigate | Compute SHA-256 from read bytes and store it in `ReplaySource` before parsing. |
| T-05-00-02 | Repudiation | parse failures | mitigate | Always write failed `ParseArtifact` JSON to `--output` when parsing reaches parser-core. |
| T-05-00-03 | Information Disclosure | stderr | mitigate | Keep stderr concise and avoid dumping full artifacts, paths beyond requested input/output, or source content. |
</threat_model>
