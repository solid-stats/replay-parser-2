<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| line coverage -> behavior confidence | Fault reporting must catch semantic mistakes coverage cannot show. |
| generated mutation output -> release gate | Reports must be validated and not accepted blindly. |
| fallback tests -> mutation equivalence | Fallback is acceptable only if it covers named high-risk fault classes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Repudiation | fault report | mitigate | Report schema records target, risk, outcome, evidence, and non-applicable rationale. |
| T-05-04-02 | Tampering | high-risk survivors | mitigate | Validator fails high-risk `missed` cases unless explicitly accepted. |
| T-05-04-03 | Denial of Service | mutation runtime | mitigate | Script uses timeout and deterministic fallback for local execution. |
</threat_model>
