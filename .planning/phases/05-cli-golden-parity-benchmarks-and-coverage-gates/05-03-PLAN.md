| coverage report -> release decision | Release gate must not pass with silent reachable production gaps. |
| allowlist -> uncovered code | Exclusions are review exceptions, not a way to hide untested behavior. |
| tests -> implementation | Tests must validate observable behavior, not private implementation trivia. |

| T-05-03-02 | Tampering | allowlist | mitigate | Machine validator rejects empty reasons, blanket exclusions, and missing inline rationale markers. |
