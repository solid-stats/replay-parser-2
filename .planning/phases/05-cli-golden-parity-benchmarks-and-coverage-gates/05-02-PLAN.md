<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| saved old artifact -> harness | Old JSON can be structurally different and must not crash comparison. |
| mismatch report -> downstream apps | Reports describe server/web impact but do not change adjacent apps. |
| old baseline evidence -> semantic truth | Current-vs-regenerated drift remains human review until approved. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Repudiation | mismatch reports | mitigate | Require mismatch category and all four impact dimensions for every finding. |
| T-05-02-02 | Tampering | baseline classification | mitigate | Keep unexplained old/current drift as `human_review` and record baseline profile. |
| T-05-02-03 | Information Disclosure | generated reports | mitigate | Keep bulky/full-corpus reports under ignored generated paths; committed reports are compact and intentional. |
</threat_model>
