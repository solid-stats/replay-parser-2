<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RabbitMQ broker to worker | Deliveries may be malformed, redelivered, or lost on connection failure until acked. |
| Worker to RabbitMQ broker | Result publication is not durable until the broker confirms it. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Repudiation | Result publish | mitigate | Use publisher confirms and treat nack/return as publish failure. |
| T-06-03-02 | Denial of Service | Consumer prefetch | mitigate | Configure `basic_qos` to default prefetch `1`. |
| T-06-03-03 | Tampering | Result routing | mitigate | Routing names are config-backed and tests assert default `parse.completed`/`parse.failed`. |
| T-06-03-04 | Denial of Service | Requeue loops | mitigate | Handled bad jobs publish `parse.failed` and ack; requeue is only for inability to publish outcome. |
| T-06-03-05 | Information Disclosure | AMQP URL/logging | mitigate | AMQP helper uses redacted config output from Plan 01 and does not log credentials. |
</threat_model>
