    Define these public schema-visible types with `Debug`, `Clone`, `PartialEq`, `Serialize`, `Deserialize`, and `JsonSchema`:

    - `ParseJobMessage` with fields:
      - `job_id: String`
      - `replay_id: String`
      - `object_key: String`
      - `checksum: SourceChecksum`
      - `parser_contract_version: ContractVersion`
    - `ArtifactReference` with fields:
      - `bucket: String`
      - `key: String`
    - `ParseResultKind` with serialized values exactly `parse.completed` and `parse.failed`.
    - `ParseCompletedMessage` with fields:
      - `message_type: ParseResultKind`
      - `job_id: String`
      - `replay_id: String`
      - `parser_contract_version: ContractVersion`
      - `source_checksum: SourceChecksum`
      - `artifact: ArtifactReference`
      - `artifact_checksum: SourceChecksum`
      - `artifact_size_bytes: u64`
      - `parser: ParserInfo`
    - `ParseFailedMessage` with fields:
      - `message_type: ParseResultKind`
      - `job_id: FieldPresence<String>`
      - `replay_id: FieldPresence<String>`
      - `object_key: FieldPresence<String>`
      - `parser_contract_version: FieldPresence<ContractVersion>`
      - `source_checksum: FieldPresence<SourceChecksum>`
      - `failure: ParseFailure`
      - `parser: ParserInfo`
    - `ParseResultMessage` as a serde untagged enum with variants `Completed(ParseCompletedMessage)` and `Failed(ParseFailedMessage)`.

    Add constructors `ParseCompletedMessage::new(...)` and `ParseFailedMessage::new(...)` that set `message_type` to the correct `ParseResultKind`. Add a helper for unsupported contract versions that builds a `ParseFailedMessage` whose `failure.error_code` is exactly `unsupported.contract_version`, `failure.stage` is `schema`, and `failure.retryability` is `not_retryable`.

    Add `crates/parser-contract/tests/worker_message_contract.rs` covering:

    - valid `ParseJobMessage` JSON deserializes with all five required fields;
    - missing `job_id`, `replay_id`, `object_key`, `checksum`, or `parser_contract_version` is rejected by serde;
    - `ParseCompletedMessage::new` serializes `"message_type":"parse.completed"`, `artifact.bucket`, `artifact.key`, `artifact_checksum`, and `artifact_size_bytes`;
    - `ParseFailedMessage::new` serializes `"message_type":"parse.failed"` and preserves `FieldPresence` unknown states for missing malformed-job fields;
    - unsupported-version helper emits `unsupported.contract_version`, `schema`, and `not_retryable`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `server-2` to parser worker | RabbitMQ job JSON is untrusted input until typed contract validation succeeds. |
| Parser worker to `server-2` | Result JSON is a durable integration contract and must not drift from generated schemas. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-00-01 | Tampering | Parse job body | mitigate | Serde and JSON Schema require all five request fields with typed checksum and contract version. |
| T-06-00-02 | Repudiation | Result messages | mitigate | Completed results carry job/replay IDs, contract version, source checksum, artifact reference, artifact checksum, size, and parser info. |
| T-06-00-03 | Denial of Service | Unsupported contract versions | mitigate | Unsupported versions become non-retryable `unsupported.contract_version` failures instead of poison-message loops. |
| T-06-00-04 | Information Disclosure | Failure messages | mitigate | Failed result envelope reuses structured `ParseFailure`; no raw replay bytes or secrets are added to result messages. |
</threat_model>
