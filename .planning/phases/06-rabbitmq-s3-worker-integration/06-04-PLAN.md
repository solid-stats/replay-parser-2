    - decode `ParseJobMessage` using `serde_json`;
    - if JSON is invalid or required fields are missing, build a `ParseFailedMessage` with unknown `FieldPresence` values where unavailable, publish it, and return `DeliveryAction::Ack` after confirm;
    - if `parser_contract_version != ContractVersion::current()`, publish non-retryable `unsupported.contract_version` failed result and return `Ack`;
    - download raw bytes by `object_key`;
    - compute and compare local SHA-256 before parsing; on mismatch publish `checksum.mismatch` failed result with `ParseStage::Checksum`, retryability `not_retryable`, then return `Ack`;
    - build `ReplaySource { replay_id: Some(job.replay_id.clone()), source_file: job.object_key.clone(), checksum: Present { value: job.checksum.clone(), source: None } }`;
    - call `parser_core::public_parse_replay` with `ParserOptions::default()`;
    - if artifact status is not `failed`, serialize artifact as minified JSON plus a trailing `\n`, compute artifact SHA-256 and byte size, build deterministic artifact key, write/reuse the object, publish `ParseCompletedMessage`, and return `Ack`;
    - if artifact status is `failed`, publish `ParseFailedMessage` using the artifact `failure` payload and return `Ack`;
    - if artifact write returns conflict or S3 output error, publish `parse.failed` with `output.artifact_conflict` or `output.s3_write` when publication is possible, then return `Ack`;
    - if result publication fails for any completed or failed outcome, return `DeliveryAction::NackRequeue`.

    - malformed job JSON publishes `parse.failed`;
    - checksum mismatch publishes `parse.failed` and does not parse/write an artifact;
    - parser failure publishes `parse.failed` and does not write an artifact;
    - S3 artifact conflict publishes `parse.failed`;
    - publisher failure returns `NackRequeue`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RabbitMQ body to typed job | Body may be invalid JSON, missing required fields, or request an unsupported contract version. |
| Raw S3 bytes to parser core | Parser must see only bytes whose SHA-256 matches the job checksum. |
| Worker process to shutdown signal | Shutdown can arrive while a job is between parse, artifact write, result publish, and ack. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Tampering | Job payload | mitigate | Decode into `ParseJobMessage`; malformed payload becomes structured `parse.failed`. |
| T-06-04-02 | Tampering | Raw object bytes | mitigate | Verify SHA-256 before parsing or writing any artifact. |
| T-06-04-03 | Repudiation | Completed result | mitigate | Completed result includes artifact reference, source checksum, artifact checksum, size, contract version, parser info, and IDs. |
| T-06-04-04 | Denial of Service | Publish failure | mitigate | If no durable outcome can be published, delivery action is `NackRequeue`; handled failures are acked after confirmed failed result. |
| T-06-04-05 | Information Disclosure | Logs/results | mitigate | Logs include IDs and error codes but not raw bytes, full artifacts, or secrets. |
| T-06-04-06 | Denial of Service | Shutdown | mitigate | Cancellation stops new deliveries while in-flight job drains through publish confirm and ack/nack. |
</threat_model>
