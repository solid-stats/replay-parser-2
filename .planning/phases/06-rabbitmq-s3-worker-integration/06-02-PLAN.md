  <action>
    Export `checksum` and `artifact_key` modules from `parser-worker/src/lib.rs`.

    In `checksum.rs`, add:

    - `sha256_hex(bytes: &[u8]) -> String` that returns lowercase SHA-256 hex;
    - `source_checksum_from_bytes(bytes: &[u8]) -> Result<SourceChecksum, WorkerError>`;
    - `verify_source_checksum(bytes: &[u8], expected: &SourceChecksum) -> Result<(), WorkerFailureKind>` where mismatch maps to a typed checksum mismatch failure, not a panic.

    In `artifact_key.rs`, add:

    - `artifact_key(prefix: &str, replay_id: &str, source_checksum: &SourceChecksum) -> Result<String, WorkerError>`;
    - default format exactly `{normalized_prefix}/{encoded_replay_id}/{source_sha256}.json`;
    - prefix normalization that trims leading/trailing `/` and rejects an empty prefix;
    - percent encoding for `replay_id` using allowed bytes `A-Z`, `a-z`, `0-9`, `-`, `_`, and `.` only;
    - rejection for empty `replay_id`;
    - no raw `/`, `\\`, standalone `.` segment, or standalone `..` segment in the encoded replay-id path segment.

    Add tests proving:

    - `artifact_key("artifacts/v3", "replay-0001", checksum)` returns `artifacts/v3/replay-0001/{sha256}.json`;
    - `replay/id`, `../replay`, `.`, `..`, spaces, and non-ASCII replay IDs are encoded or rejected so the result contains no raw traversal segment;
    - the source checksum value, not artifact checksum, is used in the deterministic key;
    - checksum helper returns the known SHA-256 of `b"abc"` as `ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad`.
  </action>
  <verify>

  </verify>

  <action>
    Implement a storage abstraction that can be tested without live S3:

    - `DownloadedObject { bytes: Vec<u8>, checksum: SourceChecksum }`
    - `ArtifactWrite { reference: ArtifactReference, checksum: SourceChecksum, size_bytes: u64, reused_existing: bool }`
    - an async `ObjectStore` trait or generic storage interface with methods for `download_raw(object_key)`, `write_artifact_if_absent_or_matching(key, bytes)`, and any minimal metadata calls required by the implementation.

    Implement `S3ObjectStore` using `aws-sdk-s3`:

    - build the client from `WorkerConfig` with `endpoint_url` only when configured, `region`, and `force_path_style` from config;
    - download raw object bytes by configured bucket and job `object_key`;
    - compute local SHA-256 after download and return it in `DownloadedObject`;
    - write artifact bytes to configured bucket/key with content type `application/json`;
    - if the artifact key already exists, read the stored bytes and compute SHA-256; reuse only when stored byte length and SHA-256 equal the new bytes;
    - if an existing object differs, return a structured worker error that later maps to `output.artifact_conflict`.

    Add no-network fake storage tests proving:

    - successful raw download returns bytes and computed SHA-256;
    - raw checksum mismatch is detectable by `verify_source_checksum`;
    - new artifact write returns `reused_existing=false`, expected key, artifact SHA-256, and byte size;
    - matching existing object returns `reused_existing=true`;
    - differing existing object returns an `output.artifact_conflict`-mappable error;
    - object-not-found and get/put errors are retryable or unknown output/input failures, not panics.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| S3 raw object to parser worker | Raw bytes can be missing, corrupt, stale, or mismatched against job checksum. |
| Parser worker to S3 artifacts | Artifact writes must be durable and deterministic without silently overwriting mismatched content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | Raw replay object | mitigate | Compute local SHA-256 and compare to job checksum before parsing. |
| T-06-02-02 | Tampering | Artifact key construction | mitigate | Percent-encode replay IDs and reject empty/ambiguous path segments. |
| T-06-02-03 | Repudiation | Artifact proof | mitigate | Return artifact key, exact artifact SHA-256, and exact byte size for completed result messages. |
| T-06-02-04 | Tampering | Existing artifact collision | mitigate | Reuse existing object only when bytes size and checksum match; otherwise return structured conflict failure. |
| T-06-02-05 | Information Disclosure | S3 errors/logs | mitigate | Storage errors include bucket/key context but not AWS credentials or signed URLs. |
</threat_model>
