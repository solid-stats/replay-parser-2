## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment and CLI to worker config | Deployment-provided values may be missing, malformed, or secret-bearing. |
| CLI binary to worker runtime | `parser-cli` should invoke worker mode without absorbing AMQP/S3 behavior into existing parse/schema/compare paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Information Disclosure | Worker config/debug logs | mitigate | Redacted debug output hides AMQP credentials and never stores AWS secret values. |
| T-06-01-02 | Denial of Service | Prefetch config | mitigate | Default prefetch is `1`; validation rejects `0`; Phase 7 owns higher-concurrency proof. |
| T-06-01-03 | Tampering | Deployment topology | mitigate | Exchange, queue, routing keys, endpoint, bucket, prefix, and path-style are config, not hardcoded deployment truth. |
| T-06-01-04 | Elevation of Privilege | Parser core boundary | mitigate | AMQP/S3/Tokio runtime dependencies are confined to `parser-worker`; parser-core remains transport-free. |
</threat_model>
