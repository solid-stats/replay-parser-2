<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Container runtime to host | The worker image runs untrusted replay inputs and external network clients in a container. |
| Smoke infrastructure to committed repository | Smoke logs and local credentials must not become committed artifacts. |
| Timeweb provider behavior to worker S3 assumptions | S3-compatible behavior may differ from AWS for conditional headers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Elevation of Privilege | Docker runtime | mitigate | Preserve non-root `USER 65532:65532` and slim runtime image. |
| T-07-03-02 | Denial of Service | Container readiness | mitigate | Docker health check targets `/readyz`, and smoke verifies both workers become ready. |
| T-07-03-03 | Repudiation | Multi-worker smoke | mitigate | Smoke asserts worker IDs and structured logs for both worker containers. |
| T-07-03-04 | Information Disclosure | Timeweb smoke | mitigate | Script requires env-supplied credentials and prints capability labels only, never secret values. |
</threat_model>
