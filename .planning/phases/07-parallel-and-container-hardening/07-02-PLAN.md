<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Worker runtime to operator logs | Logs become the main forensics surface for unattended worker containers. |
| Secret-bearing config to log events | Runtime config contains AMQP URLs and credential environment, but logs must not expose secrets. |
| Dynamic job data to telemetry cardinality | Job/replay/object identifiers are fields, not event names. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Repudiation | Worker logs | mitigate | Stable event names and worker/job/artifact/result/ack fields create an auditable lifecycle trail. |
| T-07-02-02 | Information Disclosure | Config/log output | mitigate | Redaction tests and secret grep reject AMQP passwords and AWS secret env values. |
| T-07-02-03 | Denial of Service | Log cardinality | mitigate | Event constants are low-cardinality snake_case; job/replay IDs stay in fields. |
| T-07-02-04 | Tampering | Artifact lifecycle visibility | mitigate | Artifact write/reuse/conflict decisions are logged with checksum/size context but not artifact bytes. |
</threat_model>
