<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Orchestrator to worker probe | Container scheduler uses probe results to decide whether to route work or restart the process. |
| Probe request to runtime state | External HTTP callers can poll health endpoints. |
| Dependency outage to liveness | RabbitMQ/S3 outages must not trigger liveness restarts by default. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Spoofing | Worker identity | mitigate | `worker_id` is explicit env/flag or hostname fallback and is returned in probes/logs for operator correlation. |
| T-07-01-02 | Denial of Service | Probe handlers | mitigate | Probe handlers read cached state only; dependency checks do not run per request. |
| T-07-01-03 | Information Disclosure | Probe response | mitigate | Probe JSON contains state/reason/worker ID only, no credentials, AMQP URLs, or S3 secret values. |
| T-07-01-04 | Denial of Service | Dependency outage | mitigate | RabbitMQ/S3 failures affect readiness, while liveness stays true unless fatal internal state is set. |
</threat_model>
