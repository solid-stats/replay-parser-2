  <action>
    Replace the minimal diagnostic policy from Plan 03-01 with a real accumulator:

    ```rust
    pub enum DiagnosticImpact {
        Info,
        NonLossWarning,
        DataLoss,
    }

    pub struct DiagnosticAccumulator {
        limit: usize,
        diagnostics: Vec<Diagnostic>,
        omitted_count: usize,
        has_data_loss: bool,
    }
    ```

    Required methods:

    - `new(limit: usize) -> Self`
    - `push(&mut self, diagnostic: Diagnostic, impact: DiagnosticImpact)`
    - `finish(self, context: &SourceContext) -> DiagnosticReport`

    `DiagnosticReport` should expose:

    - `diagnostics: Vec<Diagnostic>`
    - `status_for_successful_parse: ParseStatus`

    Behavior:

    - If `impact == DiagnosticImpact::DataLoss`, set `has_data_loss = true`.
    - If `diagnostics.len() < limit`, keep the diagnostic.
    - If the limit is exceeded, increment `omitted_count`.
    - `finish` appends one summary diagnostic with:
      - `code: "diagnostic.limit_exceeded"`
      - `severity: DiagnosticSeverity::Warning`
      - `message` containing the omitted count as decimal text
      - `json_path: Some("$".to_string())`
      - `expected_shape: None`
      - `observed_shape: None`
      - `parser_action: "summarized_repeated_diagnostics"`
      - `source_refs` with rule ID `diagnostic.limit_exceeded`
    - If `has_data_loss` is true, `status_for_successful_parse` is `ParseStatus::Partial`; otherwise it is `ParseStatus::Success`.

    Update metadata and entity normalization to classify drift-caused unknowns, skipped malformed entity rows, missing entity source refs, and skipped entity sections as `DiagnosticImpact::DataLoss`.

    Update artifact assembly to use `DiagnosticAccumulator::finish` for final diagnostics and status after a root object is decoded.
  </action>
  <verify>
    <automated>rg -n "DiagnosticAccumulator|DiagnosticImpact|diagnostic\\.limit_exceeded|summarized_repeated_diagnostics" crates/parser-core/src && cargo check -p parser-core --all-targets && git diff --check</automated>
  </verify>
  <acceptance_criteria>

    - `diagnostics.rs` contains diagnostic code `"diagnostic.limit_exceeded"`.
    - Metadata drift is pushed with `DiagnosticImpact::DataLoss`.
    - Malformed entity rows are pushed with `DiagnosticImpact::DataLoss`.
    - Artifact assembly sets status from `DiagnosticReport.status_for_successful_parse`.
