    Extend `raw.rs` with helpers for entries inside `$.entities`:

    - `entity_id(entity, index) -> RawField<i64>` reading numeric `id`
    - `entity_type(entity, index) -> RawField<String>` reading string `type`
    - `entity_name(entity, index) -> RawField<String>` reading string `name`
    - `entity_class(entity, index) -> RawField<String>` reading string `class`, then fallback string `_class`
    - `entity_side(entity, index) -> RawField<String>` reading string `side`
    - `entity_group(entity, index) -> RawField<String>` reading string `group`
    - `entity_description(entity, index) -> RawField<String>` reading string `description`
    - `entity_is_player(entity, index) -> RawField<bool>` accepting JSON booleans and numbers `0`/`1`
    - `entity_has_positions(entity, index) -> bool`

    Create `crates/parser-core/src/entities.rs` and implement:

    ```rust
    pub fn normalize_entities(
        raw: &RawReplay<'_>,
        context: &SourceContext,
        diagnostics: &mut DiagnosticAccumulator,
    ) -> Vec<ObservedEntity>
    ```

    Required behavior:

    - If `entities` is absent, return `Vec::new()` and emit warning diagnostic code `"schema.entities_absent"` with parser action `"skip_entities"`.
    - If `entities` is present but not an array, return `Vec::new()` and emit warning diagnostic code `"schema.entities_shape"` with parser action `"skip_entities"`.
    - For each object entity with numeric `id`, normalize an `ObservedEntity`.
    - For `type == "unit"`, set `kind: EntityKind::Unit`.
    - For `type == "vehicle"` and class `"static-weapon"`, set `kind: EntityKind::StaticWeapon`.
    - For `type == "vehicle"` and any other class, set `kind: EntityKind::Vehicle`.
    - For unknown type, set `kind: EntityKind::Unknown` and emit diagnostic code `"schema.entity_type_unknown"`.
    - For unit `identity.nickname`, use the same value as `observed_name`.
    - For vehicle/static entities, set `identity.nickname`, `identity.steam_id`, `identity.faction`, `identity.group`, `identity.squad`, `identity.role`, and `identity.description` to explicit unknown or not-applicable states as appropriate; do not invent player identity.
    - Map sides `EAST`, `WEST`, `GUER`, `CIV`, and `UNKNOWN` to `EntitySide` values. Unknown casing or malformed side becomes `EntitySide::Unknown` plus diagnostic.
    - Populate `source_refs` with a non-empty `SourceRefs` containing at least the original `$.entities[<index>]` path and entity ID. If `positions` exists, include an additional source ref with `json_path: "$.entities[<index>].positions"` so position evidence is auditable without embedding raw positions.
    - Sort final entities by `source_entity_id`, then by `kind`, then by present `observed_name`, then by present `observed_class`, then by first source path.

    Update `artifact.rs` so valid root objects call `normalize_entities` and assign the resulting vector to `artifact.entities`.
  </action>
