    In `crates/parser-core/src/artifact.rs`, implement `parse_replay(input: ParserInput<'_>) -> ParseArtifact` with this Phase 3 Plan 01 behavior:

    - Attempt `serde_json::from_slice::<serde_json::Value>(input.bytes)`.
    - If JSON decoding fails, return a failed `ParseArtifact` with:
      - `contract_version: ContractVersion::current()`
      - `parser: input.parser`
      - `source: input.source`
      - `status: ParseStatus::Failed`
      - `produced_at: None`
      - `diagnostics: Vec::new()`
      - `replay: None`
      - `entities: Vec::new()`
      - `events: Vec::new()`
      - `aggregates: AggregateSection::default()`
      - `failure.stage: ParseStage::JsonDecode`
      - `failure.error_code: ErrorCode::new("json.decode")`
      - `failure.retryability: Retryability::NotRetryable`
      - `failure.source_cause` present with the serde error string
      - `failure.source_refs` containing one source ref with `json_path: "$"` and `rule_id: "failure.json.decode"`
      - `extensions: BTreeMap::new()`
    - If JSON decoding succeeds but the root is not an object, return a failed artifact with:
      - `failure.stage: ParseStage::Schema`
      - `failure.error_code: ErrorCode::new("schema.root_object")`
      - `failure.source_cause` present value `"OCAP replay root must be a JSON object"`
      - `failure.source_refs[0].json_path == "$"`
      - `failure.source_refs[0].rule_id == "failure.schema.root_object"`
    - If the root is an object, return a success artifact shell with `replay: None`, `entities: []`, `events: []`, default aggregates, no diagnostics, no failure, and `produced_at: None`. Later plans fill metadata/entities.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted replay bytes -> parser-core | Malformed input must not panic or produce ambiguous output. |
| parser-core -> adapters | The core must not own file, queue, S3, DB, or timestamp side effects. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Denial of Service | JSON decode shell | mitigate | Return structured `json.decode` failures for invalid bytes and keep no panics/unwraps. |
| T-03-01-02 | Repudiation | failure artifact | mitigate | Include non-empty source refs and stable rule IDs for root failures. |
| T-03-01-03 | Tampering | adapter boundary | mitigate | Keep parser-core pure; no filesystem, RabbitMQ, S3, DB, or wall-clock timestamp writes. |
</threat_model>
