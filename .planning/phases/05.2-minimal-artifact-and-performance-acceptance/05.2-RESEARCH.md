# Phase 05.2 Research - Minimal Artifact and Performance Acceptance

## User Constraints

### Locked Decisions From CONTEXT.md

- D-01 through D-08: default `parse` output becomes a minimal, minified, flat statistics artifact with `players[]`, `player_stats[]`, `kills[]`, `destroyed_vehicles[]`, and `diagnostics[]`, plus only required envelope/source/replay metadata. Default rows must not include frame, time, event index, JSON path, source refs, rule IDs, normalized event snapshots, or entity snapshots. [VERIFIED: `05.2-CONTEXT.md`]
- D-09 through D-10: GitHub issue #13 vehicle score is removed from v1 implementation, contract types, schema, examples, tests, docs, and acceptance criteria. Ordinary `vehicleKills`, `killsFromVehicle`, weapon, attacker vehicle, and destroyed-vehicle facts remain. [VERIFIED: `05.2-CONTEXT.md`]
- D-11 through D-14: full normalized detail is available only through an explicit debug sidecar such as `--debug-artifact <path>`. The debug sidecar is internal, deterministic, and tested, but not the server-facing contract; default parsing must not build it unless requested. [VERIFIED: `05.2-CONTEXT.md`]
- D-15 through D-21: benchmark evidence must capture old/new baseline first, then prove x3 on a deterministic large selected replay and x10 sequentially across every file in `~/sg_stats/raw_replays`, with zero failures/skips unless an explicit allowlist is approved. Artifact-size acceptance is median artifact/raw ratio <= 5% and p95 <= 10%. [VERIFIED: `05.2-CONTEXT.md`]
- D-22: product-owner compatibility acceptance must record that `server-2` will adapt later to the minimal flat artifact while parser boundaries remain intact. [VERIFIED: `05.2-CONTEXT.md`]

## Architecture Recommendations

1. **Compatibility note first**: create `05.2-SERVER-COMPATIBILITY.md` before contract code changes. It should state that `server-2` will adapt later to minimal flat artifacts and that parser still owns no canonical identity, persistence, APIs, UI, discovery, or payout behavior.
2. **Contract v3 cutline**: bump the breaking public artifact contract from `2.0.0` to `3.0.0`; rename committed schema/examples to `v3`; keep `ParseStatus`, `ParseFailure`, `ReplaySource`, parser metadata, replay metadata, diagnostics, `side_facts`, failure invariants, and deterministic ordering.
3. **Minimal tables**: define typed rows in `crates/parser-contract/src/minimal.rs` or a replacement `compact.rs` module:
   - `MinimalPlayerRow`: `player_id`/`source_entity_id`, observed name, side, group, role, SteamID, compatibility key when needed.
   - `MinimalPlayerStatsRow`: player reference plus integer counters only: kills, deaths, teamkills, suicides/null deaths where modeled, `vehicleKills`, `killsFromVehicle`.
   - `MinimalKillRow`: killer/victim player refs where available, classification (`enemy_kill`, `teamkill`, `suicide`, `null_killer`, `unknown`), weapon, attacker vehicle context, victim side/killer side where needed, bounty eligibility/exclusion as compact enum or booleans.
   - `MinimalDestroyedVehicleRow`: killer player ref, enemy/friendly/unknown-side classification, weapon, attacker vehicle context, destroyed entity type/name/class/side where available.
4. **Internal rich model, default minimal output**: keep normalized `ObservedEntity` and `NormalizedEvent` internally for correctness, but build default minimal rows directly from that internal model and avoid serializing audit-heavy contribution/source fields. The default path must not allocate/build a full normalized debug artifact unless `ParserOptions` requests it.
5. **Debug sidecar**: add a separate debug artifact builder that emits the current rich normalized detail and source refs under explicit CLI `--debug-artifact`. Keep it deterministic and internal, with tests proving default output lacks debug-only coordinates/provenance and debug output includes them.
6. **Vehicle-score removal**: remove issue #13 score weights, contribution value types, projection keys, tests, schema definitions, fault targets, and README/planning assertions from v1. Keep neutral vehicle context names/classes/categories only when they are not issue #13 scoring concepts.
7. **Benchmark truthfulness**: split benchmark evidence into selected large replay and all-raw corpus. The selected gate should record old wall time, new wall time, speedup, selected path, raw bytes, artifact bytes, and x3 pass/fail. The all-raw gate should record every attempted raw path, success/failure/skip counts, old sequential baseline command/profile, new sequential wall time, x10 pass/fail, median/p95 artifact ratios, and failure triage.

## Don't Hand-Roll

- Do not write custom JSON tokenizers for Phase 5.2. Continue using typed Serde plus borrowed `RawValue` selective access from `raw_compact.rs`.
- Do not keep issue #13 code behind a hidden flag in the v1 parser. The locked decision is removal from v1 implementation, not hiding from the default artifact.
- Do not make the debug sidecar part of the server-facing contract. It is internal tooling and must not influence ordinary worker artifact size/performance.
- Do not silently skip raw files in the all-corpus gate. A missing, malformed, unsupported, failed, or skipped file is counted and blocks unless an explicit allowlist artifact exists.

## Common Pitfalls

| Pitfall | Mitigation |
|---------|------------|
| Minimal artifact still contains `FieldPresence`, source refs, rule IDs, frame/time, or event indexes through reused compact types. | Introduce distinct minimal row types and add tests/grep gates for forbidden default keys: `source_refs`, `rule_id`, `frame`, `event_index`, `json_path`, `events`, `entities`, `aggregate_contributions`, and `vehicle_score`. |
| Removing vehicle score breaks ordinary `vehicleKills` or `killsFromVehicle`. | Preserve vehicle context and counter tests in aggregate/minimal-table tests while deleting only issue #13 weight/projection logic. |
| Debug sidecar slows normal parse. | Add `ParserOptions`/CLI tests showing default `parse` does not build or write debug files and benchmark uses default options only. |
| Contract v3 schema/examples drift. | Keep schema generated from `parse_artifact_schema()`, update example paths and tests to `v3`, and byte-compare committed schema with fresh generation. |
| Full-corpus script measures replay-list rows instead of every raw file. | Iterate `find "$RAW_REPLAYS_DIR" -type f -name '*.json' | sort` and record attempted count equal to raw file count. |
| Performance scripts claim x10 without old sequential baseline. | Require `old_baseline_profile` containing `WORKER_COUNT=1`, old command evidence, and fail/unknown when unavailable. |
