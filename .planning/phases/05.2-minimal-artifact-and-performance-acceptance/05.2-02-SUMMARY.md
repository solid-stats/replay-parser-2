**parser-core now returns v3 minimal flat artifacts by default with retired issue 13 vehicle-score code and an explicit deterministic debug sidecar**

## Accomplishments

- Rebuilt parser-core artifact construction so `parse_replay` emits `players`, `player_stats`, `kills`, and `destroyed_vehicles` from normalized entity and combat semantics.
- Removed active issue 13 vehicle-score code and tests from parser-core while preserving ordinary vehicle destruction rows and legacy `vehicleKills`/`killsFromVehicle` counters.
- Added `DebugParseArtifact` and `parse_replay_debug` as the explicit deterministic path for full normalized entities, events, side facts, source refs, rule IDs, frames, and event indexes.
- Updated parser-core behavior tests, golden manifest entries, regression tests, and side-facts tests for the new default-vs-debug split.

## Decisions Made

- Kept full normalized entity/event/source-reference detail out of the public/default `ParseArtifact`; it is available only through parser-core's explicit `parse_replay_debug`.
- Used the existing normalized semantic model as the single source for minimal row projection rather than reconstructing counters from raw events.
- Preserved ordinary vehicle context fields and counters but removed issue 13 score weights, denominator inputs, categories, and retired vocabulary.
