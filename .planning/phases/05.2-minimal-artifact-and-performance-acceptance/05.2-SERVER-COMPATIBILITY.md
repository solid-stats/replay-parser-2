# Phase 5.2 Server Compatibility Review

## Review Scope

This review records the compatibility gate for the Phase 5.2 minimal parser
artifact redesign before implementation changes the public parser contract.

Phase 5.2 is a breaking contract/schema change after the accepted Phase 5.1
compact artifact. The default parser artifact delivered by future worker
integration must become a minimal flat v1 statistics artifact, not the Phase 5.1
compact evidence artifact and not a full normalized replay dump.

## Breaking Artifact Changes

Removed Phase 5.1 default fields:

- `participants`
- `facts`
- `summaries`
- `facts.combat`
- `facts.aggregate_contributions`
- `summaries.projections`
- `vehicle_score.inputs`
- `vehicle_score.denominator_inputs`

New Phase 5.2 default fields:

- `players`
- `player_stats`
- `kills`
- `destroyed_vehicles`
- `diagnostics`

This is a breaking public parser artifact change. The minimal artifact replaces
the Phase 5.1 compact participant/fact/summary sections with flat statistics
tables intended for later `server-2` ingestion adaptation.

## server-2 Adaptation Expectation

server-2 will adapt later to the minimal flat artifact. That adaptation is not
implemented in this repository during Phase 5.2.

The expected future adaptation is for `server-2` to consume `players[]`,
`player_stats[]`, `kills[]`, `destroyed_vehicles[]`, and `diagnostics[]` from
the parser artifact reference delivered over the later RabbitMQ/S3 worker flow.
Any PostgreSQL persistence, API response shape, OpenAPI typing, canonical player
matching, aggregate publication, and bounty payout calculation remain
`server-2` responsibilities.

## Default Artifact Policy

- default JSON is minified unless `--pretty` is supplied;
- default `kills[]` and `destroyed_vehicles[]` contain no `frame`, `time`,
  `event_index`, `event_id`, `json_path`, `source_refs`, `rule_id`, normalized
  event snapshot, or entity snapshot;
- default v1 contains no GitHub issue #13 `vehicle_score` output or scoring
  weights;
- ordinary `vehicleKills`, `killsFromVehicle`, weapon, attacker vehicle, and
  destroyed vehicle/static facts remain.

The default v1 contains no GitHub issue #13 vehicle_score output. Issue #13
vehicle score is removed from v1 implementation rather than hidden behind the
minimal artifact surface.

## Debug Sidecar Policy

Detailed event/entity/source-reference evidence belongs only behind an explicit
internal debug sidecar mode, such as `--debug-artifact <path>`.

The debug sidecar may carry normalized entities, normalized events, source refs,
rule IDs, frame/time coordinates, event indexes, JSON paths, and entity snapshots
for local investigation and parity debugging. It is not the server-facing public
contract and must not be required for ordinary worker ingestion.

## Boundaries Preserved

- parser preserves observed identity only; canonical identity belongs to server-2.
- parser does not write PostgreSQL business tables.
- parser does not own public APIs, web UI behavior, replay discovery, raw object
  staging, moderation, Steam OAuth, or bounty payout calculation.
- `replays-fetcher` continues to own production replay discovery and S3 raw
  object writes.
- `web` continues to consume `server-2` APIs and generated OpenAPI types rather
  than the parser artifact directly.

## Compatibility Risks

- `server-2` may need new ingestion mapping from the Phase 5.2 minimal flat
  artifact to PostgreSQL parse result, raw event/audit, player stat, commander,
  and bounty input tables.
- Any `server-2` code or future OpenAPI schema that expects Phase 5.1
  `participants`, `facts`, `summaries`, or `vehicle_score.*` sections will need
  a coordinated update.
- Debug-sidecar-only source refs, rule IDs, frame/time coordinates, event
  indexes, and entity snapshots are intentionally unavailable in the default
  worker artifact; downstream audit flows that need them must use explicit debug
  output or raw replay reprocessing.
- Because only brief-level downstream evidence is available locally, proceeding
  to Plan 01 requires explicit user acceptance rather than inferred
  compatibility.
