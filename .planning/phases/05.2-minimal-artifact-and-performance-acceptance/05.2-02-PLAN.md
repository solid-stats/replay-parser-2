  <action>
    Change `success_artifact` so the default `ParseArtifact` contains:

    - `players`: sorted by `source_entity_id`, including only `is_legacy_player_entity(entity)` rows. Use `player_id` format `entity:{source_entity_id}` and preserve connected/backfilled and duplicate-slot eligible players.
    - `player_stats`: one row for every `players[]` row, sorted by `source_entity_id`, with zero counters when a player has no event contributions.
    - `kills`: one row for each player death semantic from normalized killed events:
      `EnemyKill`, `Teamkill`, `Suicide`, `NullKillerDeath`, and `Unknown` when the victim is or may be a player. Do not include vehicle/static victims in `kills[]`.
    - `destroyed_vehicles`: one row for each `VehicleDestroyed` event caused by a player against vehicle or static-weapon victims, including enemy, friendly, and unknown-side cases.

    Counter mapping:

    - `kills`: increment killer for `EnemyKill`.
    - `deaths`: increment victim for `EnemyKill`, `Teamkill`, `Suicide`, `NullKillerDeath`, and player-victim `Unknown`.
    - `teamkills`: increment killer for `Teamkill`.
    - `suicides`: increment victim for `Suicide`.
    - `null_killer_deaths`: increment victim for `NullKillerDeath`.
    - `unknown_deaths`: increment victim for player-victim `Unknown`.
    - `vehicleKills`: increment attacker for `VehicleDestroyed`.
    - `killsFromVehicle`: increment killer for `EnemyKill` where `vehicle_context.is_kill_from_vehicle` is true.

    Default rows must omit these serialized keys in success artifacts: `frame`, `time`, `event_index`, `event_id`, `json_path`, `source_refs`, `rule_id`, `aggregate_contributions`, `normalized_event`, `entity_snapshot`, `vehicle_score`.
