## Existing Interfaces To Reuse

- `parser_core::parse_replay(ParserInput)` remains the pure parser entrypoint for CLI, future worker, harness, tests, and benchmarks.
- `ParserInput` and `ParserOptions` are the natural place to add an internal debug-output toggle or expose a separate debug builder entrypoint.
- `RawOcapRoot`, `compact_entities`, `compact_connected_events`, and `compact_killed_events` remain the selective source boundary.
- `normalize_entities`, `normalize_combat_events`, and `normalize_side_facts` preserve Phase 3/4 semantics and should feed minimal rows unless the executor proves a simpler direct path with equivalent tests.
- `parse_artifact_schema()` remains the schema source of truth.
- CLI tests use `assert_cmd`, temporary output files, and JSON assertions against public command behavior.
- Benchmark/fault validators use typed serializable report structs plus explicit validation errors.

## Existing Test Style

Current tests use public APIs, focused OCAP fixtures, explicit Arrange/Act/Assert comments where useful, schema/example freshness checks, deterministic serialization assertions, and command-level `assert_cmd` tests. Phase 5.2 should continue this style and strengthen it around:

- presence of `players`, `player_stats`, `kills`, `destroyed_vehicles`, and `diagnostics`;
- absence of `participants`, `facts`, `summaries`, `aggregate_contributions`, `source_refs`, `rule_id`, `frame`, `event_index`, `json_path`, and `vehicle_score` from default output;
- preservation of ordinary `killsFromVehicle`, `vehicleKills`, kill/death/teamkill/null/unknown rows, and destroyed-vehicle rows;
- explicit debug sidecar output and default-path non-contamination;
- selected large replay and all-raw benchmark validation.

## Boundary Constraints

Do not add these concerns during Phase 5.2:

- RabbitMQ/S3 worker implementation, ack/nack, or artifact upload behavior;
- PostgreSQL, API, OpenAPI, or public UI implementation;
- replay discovery or production raw fetches;
- canonical player identity matching;
- final bounty calculation or bounty payout rules;
- annual/yearly nomination product support;
- any hidden v1 issue #13 vehicle-score implementation.
