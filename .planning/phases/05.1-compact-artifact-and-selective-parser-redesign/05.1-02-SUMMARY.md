# Phase 05.1 Plan 02: Selective Parser-Core Decode Summary

## Accomplishments

- Added `raw_compact.rs` with borrowed `RawOcapRoot`, `decode_compact_root`, compact entity extraction, connected event extraction, and killed event extraction.
- Enabled `serde_json` `raw_value` in `parser-core`.
- Switched `parse_replay(ParserInput)` to call `decode_compact_root(input.bytes)` instead of decoding the root as `serde_json::Value`.
- Preserved malformed JSON failure behavior as `json.decode` and valid non-object root failure behavior as `schema.root_object`.
- Kept source coordinates for compact extraction: event index, frame, entity ID, JSON path, and rule ID propagation through existing normalization.
- Updated parser-core integration tests from removed public `entities`, `events`, and `aggregates` fields to compact `participants`, `facts.combat`, `facts.aggregate_contributions`, and `summaries.projections`.
