# Phase 05.1 Plan 06: Compact Artifact Benchmark Evidence Summary

## Accomplishments

- Extended benchmark reports with `ArtifactSizeEvidence` and `BenchmarkEvidence`, including raw input bytes, compact artifact bytes, artifact/raw ratio, selected evidence, optional whole-list/corpus evidence, and explicit whole-list unavailability rationale.
- Tightened benchmark validation so reports reject missing compact artifact evidence, invalid artifact/raw ratios, missing whole-list/corpus evidence or reason, missing RSS notes, and failed 10x status without bottleneck, parity, and artifact triage.
- Updated `benchmark-report-check` to validate and summarize the new selected/whole-list report shape.
- Renamed Criterion benchmark surfaces to `parse_only_compact_decode`, `facts_only_compact_projection`, and `end_to_end_compact_parse_replay`, and updated the projection benchmark to use compact `artifact.summaries.projections`.
- Updated `scripts/benchmark-phase5.sh --ci` to build the release CLI once, run the compact selected parse, compute raw/artifact bytes and ratio, use Markdown comparison plus JSON details when old baseline evidence is available, emit `phase: "05.1"`, and record `whole_list_unavailable_reason` unless `RUN_PHASE5_FULL_CORPUS=1` prerequisites are present.
- Updated README benchmark guidance so compact artifact size and selected plus whole-list/corpus evidence are required before a 10x claim can pass.

## Files Modified

- `crates/parser-harness/src/benchmark_report.rs` - new compact benchmark evidence schema and validation rules.
- `crates/parser-harness/tests/benchmark_report.rs` - tests for selected+whole-list, selected+unavailable-reason, missing artifact size, missing whole-list evidence, invalid ratio, RSS note, and failed 10x triage.
- `crates/parser-harness/benches/parser_pipeline.rs` - compact benchmark names and compact projection access.
- `crates/parser-harness/src/bin/benchmark-report-check.rs` - new-shape validation output.
- `scripts/benchmark-phase5.sh` - Phase 5.1 report generation, compact artifact-size metrics, Markdown+JSON comparison outputs, and optional full-corpus mode.
- `README.md` - benchmark evidence requirements for compact artifact size and whole-list/corpus scope.

## Deviations

- The default CI path does not run the whole corpus. It records the concrete reason `RUN_PHASE5_FULL_CORPUS not enabled`, and the validator allows this only as explicit unavailable evidence.

## Remaining Risks

- The generated default CI benchmark report still records selected `ten_x_status=unknown` and `parity_status=not_run` when old baseline prerequisites are not used.
- Whole-list/corpus parsing is available behind `RUN_PHASE5_FULL_CORPUS=1`, but it remains an expensive local/manual path and was not run during this plan.
- The selected compact fixture currently has an artifact/raw ratio greater than 1 because the focused fixture is tiny; this is recorded as evidence, not a passing compactness claim.
