## Standard Stack

- Keep Rust 2024, `serde`, `serde_json`, `schemars`, `semver`, `clap`, Criterion, hyperfine-compatible shell scripting, and existing workspace lint gates. [VERIFIED: `Cargo.toml`, README]
- Use `serde_json::value::RawValue` as the preferred selective parsing primitive for large top-level OCAP arrays because it can borrow the source range for a valid JSON value and avoid parsing fields until needed; docs.rs notes it requires the `raw_value` feature and can borrow through `from_slice`/`from_str`. [CITED: docs.rs serde_json RawValue]
- Treat `serde_json::StreamDeserializer` as useful for multiple adjacent JSON values, byte-offset handling, or follow-on tooling, but not sufficient by itself for a single large OCAP object without a root extractor. [CITED: docs.rs serde_json StreamDeserializer]
- Keep Criterion for stage-level Rust benchmarks and command-level shell/hyperfine-compatible reporting for old/new comparisons. Criterion docs use `cargo bench`; hyperfine supports warmups, multiple runs, outlier handling, and JSON/Markdown exports. [CITED: Criterion book; CITED: hyperfine README]

## Architecture Patterns

1. **Compatibility gate before breaking code**: write `.planning/phases/05.1-.../05.1-SERVER-COMPATIBILITY.md` first. It should list artifact fields removed/added, how `server-2` can ingest compact facts, what remains outside parser, and whether actual adjacent app evidence exists. If only briefs exist, execution should pause for user acceptance before contract implementation.
2. **Compact contract cutline**: keep the `ParseArtifact` envelope, status/failure invariants, source, diagnostics, replay metadata, parser info, contract version, `produced_at`, and side facts. Replace full `entities`/`events` with compact participant refs and compact fact/contribution sections.
3. **Internal rich semantics, compact external artifact**: parser-core may still use internal typed observations/events while deriving facts, but the serialized default artifact must not expose full normalized event/entity dumps.
4. **Selective root decode**: parse top-level scalars directly and borrow heavy `entities`/`events` array JSON through `RawValue`; deserialize only needed entity fields and relevant event tuple shapes. Use `IgnoredAny` or typed DTOs to skip large irrelevant fields such as position history.
5. **Contribution-first audit**: preserve source refs, rule IDs, contribution IDs, bounty eligibility/exclusion reasons, vehicle score matrix/applied weights, and denominator eligibility. Do not calculate final bounty or cross-replay vehicle score.
6. **Summary-first parity**: default comparison output should be Markdown with counts by category/impact, top actionable mismatches, and next action. Structured JSON remains available as detail evidence, not the default human review surface.
7. **Benchmark truthfulness**: reports must distinguish selected single-replay evidence from whole-list/corpus evidence, record unavailable corpus rationale explicitly, and block 10x pass if parity is unresolved.

## Don't Hand-Roll

- Do not build custom JSON tokenization before exhausting typed Serde plus `RawValue`; custom token scanners are higher risk and harder to prove under 100% coverage. [VERIFIED: codebase uses Serde; CITED: serde_json RawValue docs]
- Do not hand-roll final bounty point calculation in parser-core; `server-2` owns cross-replay/previous-rotation bounty state. [VERIFIED: `gsd-briefs/server-2.md`, phase CONTEXT.md]
- Do not add RabbitMQ/S3 worker behavior in Phase 5.1; worker mode depends on the compact artifact and belongs to Phase 6. [VERIFIED: ROADMAP]
- Do not make comparison JSON the only review surface; Phase 5 UAT rejected unreadable comparison output. [VERIFIED: STATE.md, phase CONTEXT.md]

## Common Pitfalls

| Pitfall | Mitigation |
|---------|------------|
| Contract tests still expect `entities` or `events` because schema/examples were not refreshed. | Plan schema/example updates and tests that assert those top-level fields are absent from success and failure examples. |
| Parser rewrite drops Phase 3/4 edge-case behavior. | Re-run and strengthen tests for metadata drift, connected-player backfill, duplicate-slot same-name compatibility, null killer, teamkill, vehicle score clamp, commander/outcome, deterministic output, and source refs. |
| "Selective" parser still calls `serde_json::from_slice::<Value>` in normal code. | Add grep gates and tests for `crates/parser-core/src/artifact.rs` and normal raw modules; allow `Value` only in contract payloads, tests, harness reports, or explicit debug tooling. |
| Compact artifact hides audit evidence needed by `server-2`. | Include participant refs, combat facts, aggregate contribution refs, source refs, rule IDs, and exclusion reasons. |
| Whole-corpus evidence silently degrades to one replay. | Benchmark report validator must require selected evidence and a whole-list/corpus evidence object or explicit unavailable rationale. |
| Cross-app risk is under-recorded. | First execution plan writes compatibility review and blocks if actual `server-2` contract evidence is missing. |
