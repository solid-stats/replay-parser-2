# Phase 05.1 Plan 05: Summary-First Comparison Reports Summary

## Accomplishments

- Added a comparison review summary model and Markdown renderer with category counts, impact counts, prioritized top diffs, and next-action text.
- Made `replay-parser-2 compare` write Markdown by default while preserving explicit structured JSON through `--detail-output` and `--format json`.
- Rejected `--format json` combined with `--detail-output` because `--output` already receives the detailed JSON report in that mode.
- Updated selected comparison surfaces from legacy full-output paths to compact artifact paths: `status`, `replay`, `participants`, `facts.combat`, `facts.aggregate_contributions`, and compact `summaries.projections.*` keys.
- Added harness and CLI tests proving default Markdown starts with `# Comparison Summary`, JSON detail contains `findings`, and `human_review` reports carry the required next action.

## Files Modified

- `crates/parser-harness/src/summary_report.rs` - new summary model and Markdown renderer.
- `crates/parser-harness/src/lib.rs` - exports the summary report module.
- `crates/parser-harness/src/comparison.rs` - compares compact artifact surfaces.
- `crates/parser-harness/tests/comparison_report.rs` - covers Markdown summary sections, compact surfaces, and `human_review` next-action text.
- `crates/parser-cli/src/main.rs` - adds `--format <markdown|json>`, `--detail-output <path>`, Markdown default output, and invalid-combination handling.
- `crates/parser-cli/tests/compare_command.rs` - covers Markdown default output, JSON details, explicit JSON format, and invalid option combinations.
