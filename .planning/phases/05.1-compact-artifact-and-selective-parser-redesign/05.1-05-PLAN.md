    Add `crates/parser-harness/src/summary_report.rs` and export it from `lib.rs`.

    Implement:

    - `ComparisonTopDiff { surface: String, category: MismatchCategory, parser_artifact_impact: ImpactLevel, server_2_recalculation_impact: ImpactLevel, ui_visible_public_stats_impact: ImpactLevel, note: String }`
    - `ComparisonReviewSummary { total_findings: usize, by_category: BTreeMap<String, usize>, by_impact: BTreeMap<String, usize>, top_diffs: Vec<ComparisonTopDiff>, next_action: String }`
    - `ComparisonReviewSummary::from_report(report: &ComparisonReport) -> Self`
    - `render_markdown_summary(report: &ComparisonReport) -> String`

    Markdown output must include these headings:

    - `# Comparison Summary`
    - `## Counts by Category`
    - `## Counts by Impact`
    - `## Top Diffs`
    - `## Next Action`

    Select up to 10 top diffs in this priority order: `new_bug`, `human_review`, `insufficient_data`, `intentional_change`, `old_bug_fixed`, `old_bug_preserved`, `compatible`.

    If any finding is `human_review`, `next_action` must contain `Review top human_review diffs before accepting parity`.
  </action>
