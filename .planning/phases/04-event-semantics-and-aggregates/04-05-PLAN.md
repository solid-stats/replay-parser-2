  <action>
    Add `pub mod side_facts;` to parser-core `lib.rs`.

    Create `crates/parser-core/src/side_facts.rs` with:

    ```rust
    pub fn normalize_side_facts(
        raw: &RawReplay<'_>,
        entities: &[ObservedEntity],
        context: &SourceContext,
        diagnostics: &mut DiagnosticAccumulator,
    ) -> ReplaySideFacts
    ```

    Outcome behavior:

    - Check raw string candidates for keys `winner`, `winningSide`, and `outcome`.
    - Accepted side aliases:
      - `WEST`, `west`, `BLUFOR` -> `EntitySide::West`
      - `EAST`, `east`, `OPFOR` -> `EntitySide::East`
      - `GUER`, `guer`, `INDEPENDENT` -> `EntitySide::Guer`
      - `CIV`, `civilian` -> `EntitySide::Civ`
    - First accepted candidate becomes `OutcomeStatus::Known`, winner side `Present`, source label `Present`, confidence `Some(Confidence::new(1.0)?)`, rule ID `side_facts.outcome.explicit_field`, and source refs from the raw field.
    - If no accepted candidate exists, return `OutcomeStatus::Unknown`, winner side `Unknown { reason: MissingWinner }`, source label `Unknown { reason: SourceFieldAbsent }`, rule ID `side_facts.outcome.unknown`, no source refs, and do not push a diagnostic.
    - If a candidate key exists but has an unrecognized value, push diagnostic code `side_facts.outcome_unrecognized` with `DiagnosticImpact::NonLossWarning`; outcome remains unknown.

    Commander candidate behavior:

    - Find unit entities whose observed `description`, `role`, `group`, or `observed_name` contains case-insensitive `"ks"`, `"commander"`, or `"командир"`.
    - Emit one `CommanderSideFact` candidate per matching entity.
    - `kind: CommanderFactKind::Candidate`
    - `confidence: Some(Confidence::new(0.6)?)`
    - `rule_id: side_facts.commander.keyword_candidate`
    - `side` comes from entity identity side.
    - `side_name` is the lowercase side string when side is present; otherwise unknown.
    - `commander` is an `EventActorRef` built from the entity.
    - source refs include the entity source refs.
    - Missing commander candidates should return an empty commanders array and should not mark partial status.

    Do not use this heuristic to set canonical commander truth.
  </action>
  <verify>
