      - `observed_entity_ids`
      - `observed_name`
      - `side`
      - `kills`
      - `killsFromVehicle`
      - `vehicleKills`
      - `teamkills`

      - `totalPlayedGames: 1`
      - `source_contribution_ids`
    - `legacy.relationships`: object containing arrays `killed`, `killers`, `teamkilled`, `teamkillers`; each row has source/target compatibility keys, names, count, event IDs, and contribution IDs.
    - `legacy.game_type_compatibility`: object with `mission_name`, `prefix_bucket`, and `parser_action: "emit_filter_metadata_only"`. Bucket rules are `sg`, `mace`, `sm`, `sgs`, or `other` based on `mission_name` prefix.
    - `legacy.squad_inputs`: sorted rows containing compatibility key, observed name, parsed bracket squad prefix if present, source entity IDs, and source refs. Use null/unknown JSON for absent prefix; do not apply nameChanges.csv.
    - `legacy.rotation_inputs`: object with `requires_downstream_replay_date: true` and `parser_action: "server_or_parity_harness_groups_by_replay_date"`.
    - `bounty.inputs`: array containing only enemy player kill bounty inputs. Teamkill, suicide, null killer, vehicle victim, and unknown actor events must not appear in this array.

    Formula requirements:

    - `deaths.total` is `1` when a row has `isDead == true`, else `0`.
    - `deaths.byTeamkills` is `1` when a row has `isDeadByTeamkill == true`, else `0`.
    - `kdRatio` uses legacy formula with denominator `abs(deaths.total - deaths.byTeamkills)` and returns `kills - teamkills` when denominator is zero.
    - `score` uses legacy formula with denominator `totalPlayedGames - deaths.byTeamkills` and returns `kills - teamkills` when denominator is `<= 0`.
    - `killsFromVehicleCoef` is `0` when `kills == 0` or `killsFromVehicle == 0`, else rounded to 2 decimals.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| normalized events -> aggregate projections | Counters must not be invented or silently mutated. |
| parser artifact -> server recalculation | Backend must be able to audit and recalculate values from source refs. |
| parser compatibility key -> canonical identity | Compatibility grouping must not become canonical player matching. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Repudiation | aggregate counters | mitigate | Every counter/projection row carries contribution IDs linked to event/source refs. |
| T-04-03-02 | Tampering | bounty inputs | mitigate | Only enemy kill events populate `bounty.inputs`; excluded events are tested. |
| T-04-03-03 | Elevation of Privilege | identity projection | mitigate | Use observed entity IDs and `legacy_name:` keys, never canonical player IDs. |
</threat_model>
