  <action>
    In `raw.rs`, add:

    ```rust
    #[derive(Debug, Clone, PartialEq)]
    pub struct KilledEventObservation {
        pub event_index: usize,
        pub frame: u64,
        pub killed_entity_id: Option<i64>,
        pub kill_info: KilledEventKillInfo,
        pub distance_meters: Option<f64>,
        pub json_path: String,
    }

    #[derive(Debug, Clone, PartialEq, Eq)]
    pub enum KilledEventKillInfo {
        NullKiller,
        Killer { killer_entity_id: i64, weapon: Option<String> },
        Malformed { observed_shape: String },
    }
    ```

    Add `pub fn killed_events(raw: RawReplay<'_>) -> Vec<KilledEventObservation>` that scans top-level `events` only when it is an array.

    It must treat an event as a killed-event candidate only when `event[1] == "killed"`. For each candidate:

    - `event[0]` numeric frame becomes `frame`; if absent or non-numeric, skip the candidate because no stable frame coordinate exists.
    - `event[2]` numeric killed ID becomes `killed_entity_id`; otherwise set `None`.
    - `event[3] == ["null"]` becomes `KilledEventKillInfo::NullKiller`.
    - `event[3] == [killer_id, weapon]` with numeric killer ID becomes `KilledEventKillInfo::Killer`; weapon is `Some(String)` only when string and non-empty.
    - any other `event[3]` shape becomes `KilledEventKillInfo::Malformed { observed_shape }`.
    - `event[4]` numeric distance becomes `distance_meters`; otherwise `None`.
    - `json_path` is exactly `$.events[{event_index}]`.

    Do not emit diagnostics here; raw accessors only preserve observations. Semantic diagnostics are added in Plan 04-02.
  </action>
  <verify>
    <automated>rg -n "KilledEventObservation|KilledEventKillInfo|killed_events" crates/parser-core/src/raw.rs && cargo check -p parser-core --all-targets && git diff --check</automated>
  </verify>
  <acceptance_criteria>
    - `raw.rs` contains `pub struct KilledEventObservation`.
    - `raw.rs` contains `pub enum KilledEventKillInfo`.
    - `raw.rs` contains `pub fn killed_events`.
    - Null killer shape `["null"]` is represented as `KilledEventKillInfo::NullKiller`.
    - Killer shape `[killer_id, weapon]` is represented as `KilledEventKillInfo::Killer`.
    - `cargo check -p parser-core --all-targets` exits 0.
  </acceptance_criteria>
  <done>Raw killed tuple handling is isolated and tolerant.</done>
