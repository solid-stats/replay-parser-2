  <name>Task 2: Emit vehicle score contributions and denominator inputs</name>
  <files>crates/parser-core/src/aggregates.rs, crates/parser-core/src/vehicle_score.rs</files>
  <read_first>

  </read_first>
  <action>
    Extend aggregate derivation so `vehicle_score.inputs` and `vehicle_score.denominator_inputs` are populated from normalized combat events.

    Contribution creation rules:

    - Only events with `combat.vehicle_context.is_kill_from_vehicle == true` are eligible for vehicle score.
    - Enemy vehicle kill/player kill from vehicle creates `AggregateContributionKind::VehicleScoreInput` with `VehicleScoreSign::Award`.
    - Same-side non-suicide kill from vehicle creates `AggregateContributionKind::VehicleScoreInput` with `VehicleScoreSign::Penalty`.
    - Suicide, null-killer, unknown actor, non-vehicle weapon, unknown attacker category, or unknown target category creates no vehicle-score contribution.
    - Contribution ID format is `aggregate.vehicle_score.{event_id}`.
    - Rule ID for award is `aggregate.vehicle_score.award`.
    - Rule ID for penalty is `aggregate.vehicle_score.penalty`.
    - Contribution value must use `VehicleScoreInputValue`.
    - Penalty contributions must set:
      - `matrix_weight` to the raw matrix value
      - `applied_weight` to `max(matrix_weight, 1.0)`
      - `teamkill_penalty_clamped` to true when applied differs from raw
    - Award contributions set `applied_weight == matrix_weight` and `teamkill_penalty_clamped == false`.

    Denominator rules:

    - `vehicle_score.denominator_inputs` is a sorted array with one row per player compatibility key that has at least one award contribution in this replay.
    - Each row has `compatibility_key`, `observed_entity_ids`, `has_vehicle_kill: true`, and `source_contribution_ids`.
    - Do not compute final cross-replay score in parser-core.
  </action>
