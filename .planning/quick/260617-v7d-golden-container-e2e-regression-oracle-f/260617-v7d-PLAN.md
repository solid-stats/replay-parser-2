Build a golden container-e2e regression oracle for `replay-parser-2`: a testcontainers-backed worker
e2e that boots ephemeral RabbitMQ + MinIO, drives the real worker through the
`run_until_cancelled` + `CancellationToken` seam, and pins the FULL observable contract byte-for-byte

artifact key + checksum, idempotency, checksum-mismatch failure). The same `*.expected.json` baseline
is reused by a fast in-process parser-core golden test so parser drift fails without containers. The
e2e is `#[ignore]`, excluded from the coverage gate, and runs as a master-only pre-deploy CI job. A
deterministic capture script + presence skip-guards let a human add real `~/sg_stats` replays later
without breaking `verify`.

Purpose: the v1.1 behavior-preserving refactor (Phases 9-12) rides on this oracle — any observable
drift the refactor could introduce turns it red before deploy. It complements (does not replace) the
fast parser-core golden suite.

Output: `golden_container_e2e.rs` + shared `tests/common/mod.rs`, the committed `*.expected.json`
baseline, a fast `golden_artifact_bytes.rs` consumer, `scripts/capture-golden-replays.sh`, a
master-only CI job in `cd.yml`, and `TEETH-PROOF.md` recording the mutation runs.
